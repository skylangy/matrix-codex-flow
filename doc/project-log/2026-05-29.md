## 已发布：第二优先级

第二优先级已经发布到 GitHub：

- PR：`https://github.com/skylangy/matrix-codex-flow/pull/2`
- main 合并提交：`b2e40cd`

### 背景问题

第一优先级完成后，Codex 会话连续性和重复消息问题已经修复并发布。但默认执行策略仍然偏激进：

- 后端默认使用 `DangerFullAccess`。
- 后端默认开启 shell 网络访问。
- 当前应用还没有实现交互式审批 UI，因此不能直接依赖 `OnRequest` 审批模式。

### 第二优先级改动

| 模块 | 改动 |
| --- | --- |
| 前端 Agent 配置模型 | 增加 `sandboxMode` 和 `networkAccessEnabled` |
| 默认配置 | 默认 Codex Agent 改为 `workspace-write`，并关闭 shell 网络访问 |
| 设置界面 | 新增 `Execution Safety` 区域，可选择沙盒模式和网络访问 |
| Rust 请求模型 | 接收前端传入的安全配置 |
| Rust Codex 策略 | 将前端沙盒模式映射到 Codex SDK 的 `SandboxMode` |

当前仍然保留：

```rust
ApprovalMode::Never
```

原因是应用还没有实现 Codex 审批事件到用户确认 UI 的完整闭环。后续需要把交互式审批 UI 作为独立任务完成。
